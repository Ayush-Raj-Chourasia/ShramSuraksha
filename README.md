# 🛡️ ShramSuraksha — AI-Powered Parametric Insurance for India's Gig Economy

> **Team Era** | Guidewire DEVTrails 2026 | Phase 2 Submission

## 📋 Problem Statement

India's **platform-based delivery partners** (Zomato, Swiggy, Zepto, Amazon, Blinkit) are the backbone of our digital economy. However, external disruptions like **extreme weather, pollution, and curfews** cause them to lose **20–30% of monthly earnings** with **zero income protection**.

**ShramSuraksha** is an AI-enabled **parametric insurance platform** that provides automated coverage and instant payouts to gig workers when external disruptions impact their ability to work.

## 🎯 Our Persona: Food & Quick-Commerce Delivery Workers

We focus on delivery partners from **Zomato, Swiggy, Zepto, Blinkit** — workers who operate outdoors and are most vulnerable to weather and environmental disruptions.

## ⚡ Key Features

### 1. Progressive Web App (PWA) & Optimistic UI
- **Offline-Ready:** Wraps the entire application in a Service Worker (`vite-plugin-pwa`). Drivers can install it natively on Android/iOS, ensuring the app loads even in cellular dead zones.
- **TanStack React Query:** UI inputs (like "File Claim") execute with Optimistic Updates, feeling instantaneous regardless of network latency.

### 2. Upstash / Redis Caching Layer
- We implemented a Redis Key-Value caching system to wrap our OpenWeatherMap triggers. 
- Heavily mitigates API rate limiting and drastically improves latency, storing the city's weather stats centrally for all users in the same sector.

### 3. Supabase Realtime WebSockets
- Escaped traditional HTTP polling. Parametric triggers generated by the backend are pushed instantly via Supabase `postgres_changes` WebSockets to the driver's phone with zero latency.

### 4. AI-Powered Dynamic Premium Calculation
- **Gemini AI** calculates personalized weekly premiums based on:
  - Zone-specific risk factors
  - Weather patterns and claims history

### 5. Intelligent Fraud Detection
- **Duplicate checks** (blocked within 6 hours)
- **GPS tracking** vs Registered Zone mismatch logic.
- Claims evaluated by Gemini for context before automatic payouts are dispatched via Mock UPI in < 90 seconds.

## 🚀 Getting Started Locally

```bash
# Clone the repo
git clone https://github.com/Ayush-Raj-Chourasia/ShramSuraksha.git
cd ShramSuraksha

# Install frontend dependencies
npm install

# Install backend dependencies
cd server && npm install && cd ..
```

### Environment Variables

Create a `.env` file in the root:
```env
SUPABASE_URL=your_supabase_project_url
SUPABASE_SERVICE_KEY=your_supabase_secret_key
WEATHER_API_KEY=your_openweathermap_key
GEMINI_API_KEY=your_gemini_api_key
```

### Running Locally
```bash
# Terminal 1: Start backend
cd server && node index.js

# Terminal 2: Start frontend
npm run dev
```
